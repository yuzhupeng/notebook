但是API的动态调用，也查的很严

比如你多次loadlibrary，每load一次，杀软就监控你一次

然后就对你打一个印象分，最后印象不好就kill 你



所以我们有时不能频繁的去调用敏感的API，不然会被打上高危的标签

于是，如果我们重写API的话，就尽可能的规避了杀软



如果你的导入表太多的敏感API,跑起来就被杀

当然你可以导入一半不太敏感的API,动态导入一半不太敏感的API

这样会比不导入方便一些
